# 데이트앱 1차 요구사항 정리

## 목표

첫 구현 범위는 `회원가입/로그인`, `일정 공유 캘린더`, `3교대 스케줄 표시`이다.

사용자는 하나의 달력 화면에서 아래 정보를 함께 확인할 수 있어야 한다.

- 내 일정
- 우리 일정
- 상대 일정
- 근무 스케줄

이 문서는 실제 구현을 위한 MVP 기준 요구사항만 우선 정리한다.

## 핵심 기능

### 1. 회원가입 및 로그인

사용자는 카카오 계정으로 서비스에 가입하고 로그인할 수 있어야 한다.

인증 구현 기준:

- 서버 보안 프레임워크는 `Spring Security`
- 소셜 로그인 공급자는 `Kakao OAuth2`
- 앱 인증 상태는 `JWT`로 유지

최소 요구사항:

- 최초 카카오 로그인 시 자동 회원가입
- 재로그인 시 기존 계정 매핑
- 액세스 토큰 발급
- 리프레시 토큰 기반 재발급
- 로그아웃 지원

### 2. 일정 공유 캘린더

하나의 월간 캘린더 화면에서 일정 주체별로 다른 색상으로 일정을 구분해서 보여준다.

일정 주체는 아래 3가지다.

- 나
- 우리
- 상대

각 일정은 최소한 아래 정보를 가진다.

- 일정 ID
- 제목
- 시작 날짜와 시작 시간
- 종료 날짜와 종료 시간
- 일정 주체
- 색상 또는 주체 기반 표시 스타일
- 메모(선택)

### 3. 3교대 스케줄 동시 표시

캘린더 일정과 별도로, 근무 스케줄을 같은 화면에서 함께 확인할 수 있어야 한다.

지원하는 스케줄 타입은 아래 6가지다.

- Day
- Night
- Mid
- Evening
- Off
- 휴가

근무 스케줄도 날짜 단위로 등록되며, 일정과 겹쳐서 표시할 수 있어야 한다.
커플 두 명이 모두 스케줄 근무를 수행하는 경우 각자의 근무 스케줄을 별도로 저장할 수 있어야 한다.
메인 달력 화면에서는 선택을 통해 나와 상대방 중 한 명의 근무 스케줄만 일정과 함께 확인할 수 있어야 한다.
커플 중 한 명이 상근 등으로 근무 스케줄을 별도로 기록할 필요가 없는 경우, 사용자는 설정을 통해 메인 달력의 기본 근무 스케줄 표시 대상을 상대방으로 지정할 수 있어야 한다.
이 설정이 켜져 있으면 앱 진입, 월 이동, 날짜 상세 진입 시 상대방의 근무 스케줄이 기본으로 표시되어야 하며, 사용자는 필요할 때만 표시 대상을 변경할 수 있어야 한다.

## MVP 화면 요구사항

### 인증 화면

로그인되지 않은 사용자는 먼저 인증 화면에 진입해야 한다.

화면 구성:

- 간단한 서비스 소개 문구
- `카카오로 시작하기` 버튼
- 로그인 실패 시 오류 메시지

로그인 성공 후에는 월간 캘린더 화면으로 이동한다.

### 메인 화면

앱 실행 시 가장 먼저 진입하는 화면은 `월간 캘린더 화면`이다.

화면 구성:

- 상단: 현재 연/월 표시, 이전 달/다음 달 이동
- 본문: 월간 캘린더 그리드
- 하단 또는 상단 필터 영역: 표시 대상 토글
- 빠른 등록 버튼: 일정 또는 스케줄 등록

### 캘린더 셀 표시 규칙

하루 셀에는 아래 정보가 함께 보일 수 있다.

- 날짜
- 내 일정 표시 점 또는 라벨
- 우리 일정 표시 점 또는 라벨
- 상대 일정 표시 점 또는 라벨
- 근무 스케줄 배지 또는 텍스트

표시 우선순위:

1. 날짜는 항상 보여야 한다.
2. 근무 스케줄은 짧은 코드 또는 배지 형태로 보여준다.
3. 일정은 색상 점 또는 최대 1~2개 요약 라벨로 보여준다.
4. 일정이 많은 경우 `+N` 형태로 축약한다.

## 등록 기능 요구사항

### 일정 등록

사용자는 일정 등록 시 아래 항목을 입력할 수 있어야 한다.

- 제목
- 시작 날짜와 시작 시간
- 종료 날짜와 종료 시간
- 일정 주체 선택: 나 / 우리 / 상대
- 메모(선택)

일정 추가 화면, 일정 수정 화면, 날짜 상세 화면에서는 모두 일정의 시작 날짜/시간과 종료 날짜/시간을 사용자가 확인할 수 있어야 한다.

### 스케줄 등록

사용자는 날짜별로 근무 스케줄을 등록할 수 있어야 한다.

입력 항목:

- 날짜
- 스케줄 타입 선택: Day / Night / Mid / Evening / Off / 휴가

추가 고려:

- 한 날짜에 스케줄은 1개만 허용할지
- 여러 날짜에 반복 등록을 지원할지

MVP에서는 아래 기준으로 단순화한다.

- 한 날짜에 스케줄 1개
- 반복 등록은 제외

### 이미지 기반 스케줄 일괄 등록

사용자는 Android 앱에서 본인 이름에 해당하는 월간 근무표 행 이미지를 선택해 한 달 스케줄을 자동 추출할 수 있어야 한다.

개인정보 노출을 줄이기 위해 이미지는 서버에 업로드하지 않고, 본인 스케줄 행만 선택 또는 자른 뒤 앱 내부 Google ML Kit OCR로 처리하는 방식을 기본 UX로 한다.

지원하는 근무표 코드는 아래와 같다.

- `/`: Off, 휴가 포함
- `D`: Day
- `E`: Evening
- `n`: Night
- `m`: Mid

자동 추출 결과는 즉시 저장하지 않고 월간 스케줄 수정 화면에 먼저 표시해야 한다. 사용자는 잘못 인식된 날짜를 직접 수정한 뒤 기존 월별 일괄 저장 API로 확정 등록한다.

권장 플로우:

1. 사용자가 기준 월을 선택한다.
2. 사용자가 Android 앱에서 본인 스케줄 행 이미지를 선택한다.
3. 앱이 Google ML Kit OCR로 날짜별 스케줄 코드를 추출한다.
4. 앱은 추출 결과와 누락/오인식 항목을 사용자에게 보여준다.
5. 사용자가 수정 후 확정하면 월별 일괄 저장 API를 호출한다.

## 인증 요구사항

### 카카오 로그인

사용자는 카카오 OAuth 인증을 통해 서비스에 로그인할 수 있어야 한다.

서버 동작:

- 카카오 인가 코드를 받아 사용자 식별값을 확인한다.
- 신규 사용자면 회원가입 처리한다.
- 기존 사용자면 로그인 처리한다.

### 토큰 유지

로그인 성공 후 앱은 JWT를 이용해 인증 상태를 유지할 수 있어야 한다.

- 액세스 토큰
- 리프레시 토큰
- 액세스 토큰 만료 시 재발급

### 로그아웃

사용자는 현재 기기 기준으로 로그아웃할 수 있어야 한다.

## 필터 및 보기 옵션

사용자는 캘린더에서 보고 싶은 정보만 선택해서 볼 수 있어야 한다.

필터 항목:

- 내 일정 보기
- 우리 일정 보기
- 상대 일정 보기
- 근무 스케줄 보기
- 기본 근무 스케줄 표시 대상: 내 스케줄 / 상대 스케줄

MVP에서는 기본적으로 전체 표시를 기본값으로 한다.
단, 사용자가 기본 근무 스케줄 표시 대상을 상대 스케줄로 설정한 경우에는 캘린더 진입 시 상대방 근무 스케줄을 기본 표시한다.

## 데이터 모델 초안

### User

- `id: String`
- `oauthProvider: String`
- `oauthProviderUserId: String`
- `nickname: String`
- `profileImageUrl: String?`
- `defaultShiftOwnerType: EventOwnerType`
- `createdAt: Instant`
- `updatedAt: Instant`

### Event

- `id: String`
- `title: String`
- `startDate: LocalDate`
- `endDate: LocalDate`
- `startTime: LocalTime`
- `endTime: LocalTime`
- `ownerType: EventOwnerType`
- `note: String?`

### EventOwnerType

- `ME`
- `US`
- `PARTNER`

### ShiftSchedule

- `id: String`
- `groupId: String`
- `date: LocalDate`
- `ownerUserId: String`
- `shiftType: ShiftType`

### ShiftType

- `DAY`
- `NIGHT`
- `MID`
- `EVENING`
- `OFF`
- `VACATION`

## 첫 구현 우선순위

### 1단계

회원가입/로그인 흐름 구성

- 카카오 로그인
- 토큰 발급
- 로그인 상태 유지

### 2단계

월간 캘린더 화면 뼈대 구성

- 월 이동
- 날짜 그리드
- 더미 데이터 표시

### 3단계

일정 주체별 색상 표시

- 나 / 우리 / 상대 색상 분리
- 하루 셀에 점 또는 라벨 표시

### 4단계

근무 스케줄 표시

- Day / Night / Mid / Evening / Off / 휴가 배지 표시
- 일정과 동시에 보이도록 배치

### 5단계

등록 기능 연결

- 일정 추가
- 스케줄 추가

## MVP 완료 기준

아래가 되면 1차 구현 완료로 본다.

- 사용자는 카카오 계정으로 회원가입/로그인할 수 있다.
- 로그인 상태가 JWT 기반으로 유지된다.
- 하나의 월간 캘린더 화면에서 일정과 스케줄을 함께 볼 수 있다.
- `나`, `우리`, `상대` 일정이 서로 다른 시각 요소로 구분된다.
- Day / Night / Mid / Evening / Off / 휴가 스케줄이 날짜에 연결되어 보인다.
- 특정 날짜에 일정과 스케줄을 각각 등록할 수 있다.
- 필터로 각 항목의 표시 여부를 바꿀 수 있다.
